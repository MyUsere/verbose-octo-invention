Настройка имени главной ветки в Git

git config --global init.defaultBranch master


индексировать текущее состояние нашего файла:

git add index.html


какие изменения были внесены в файлы

git diff


Чтобы наверняка удалить ненужный нам файл

git rm -f test


прекращение слежения за файлом

git rm --cached index.html


восстановление удаленного файла:

git checkout myfile.txt


переименование файла:

git mv старый новый


отмена индексации файла:

git restore --staged index.html


перемещение в другую папку:

git mv ./new.txt old/file.txt

Ветки


создание новой ветки

git branch rest


удаление ветки:

git branch -D rest


переключение на другую ветку:

git checkout rest


переименование ветки:

git branch --move старое имя новое имя


. Вливает ветку branch-name в ветку, в которой вы находитесь.

git merge branch-name


Слитые ветки:

git branch --merged


не слитые ветки:

git branch --no-merged


Коммиты


просмотр истории коммитов

git log --pretty=format:"%h - %s "

git log --pretty=format:"%h - %s : %ad [ %an ]"


Изменить сообщение коммита:

git commit --amend -m "Новое прекрасное сообщение"


возврат к предыдущему состоянию

git reset HEAD index.html


псевдонимы для команд:

git config --global alias.form log --pretty=format:"%h - %s "


Теперь для просмотра коммитов вам будет достаточно ввести в терминал:

git form



Удаленный репозиторий на GitHab


Для публикации локальной ветки в удаленный репозитарий можно использовать команду:

git push -u origin имя ветки


Для переименования ветки, загруженной на удалённый сервер, нужно вначале переименовать ветку локально, а затем выгрузить изменения обратно, удалив предыдущую ветку на сервере

git branch -m старое_название новое_название

git push origin :старое_название новое_название


Добавление внешних репозиториев в Git:

git remote add origin (адрес)


Просмотр информации внешнего репозитория в Git:

git remote show origin


Получение имени внешнего репозитория в Git:

git remote


Адреса внешнего репозитория в Git:

git remote -v


Переименование внешнего репозитория в Git:

git remote rename (старое) ( новое название)


проверка нового имени:

git remote


Удаление внешнего репозитория в Git:

git remote rm (имя репозитория)


отправим данные во внешний репозиторий:

git push -u origin master


Отправляет все зафиксированные изменения с локального репозитория в удалённый. Это одна из самых важных команд, ведь все вышеописанные действия производятся в локальной копии репозитория. Когда вы закончите работу, эту копию нужно будет отправить в удалённый репозиторий. Только так другие участники процесса смогут получить актуальную версию.

git push


используем git fetch, чтобы получить последние изменения из удаленного репозитория:

git fetch origin


Получение изменений из удалённого репозитория

git pull


Удаление удалённой ветки

git push --delete origin (имя ветки)
